本日は思考の連鎖を行い、より精度の高い推論を行うAIモデルについて説明していきます。

概要

思考の連鎖のことを CoT（Chain of Thought：思考の連鎖） と呼びます。

CoTを行うモデルとは、一言で言えば 「答えを出す前に、人間のように一歩ずつ論理を組み立ててから回答するAI」 のことです。

従来のモデルが「直感」で即答していたのに対し、CoTモデルは「熟考」してから答えるという大きな違いがあります。

1. CoTの概要：システム1からシステム2へ

心理学者のダニエル・カーネマンが提唱した「二重過程理論」になぞらえて説明されることが一般的です。

  • 従来のモデル（システム1）: 直感的、高速。単語の並びの確率から即座に答えを生成します。そのため、複雑な計算や論理パズルで「うっかりミス」をしやすい傾向がありました。
  • CoTモデル（システム2）: 論理的、遅い。問題を小さなステップに分解し、 「まずAを確認し、次にBを計算し、その結果からCを導く」 という中間プロセスを明示的に踏みます。

2. 2種類のCoTアプローチ

CoTを実現する方法には、大きく分けて「プロンプトによる手法」と「モデルの設計による手法」の2つがあります。

① プロンプトによるCoT（Prompting）

既存の汎用モデルに対し、ユーザーが指示を与えることで思考を促す手法です。

  • 有名な呪文: 「ステップバイステップで考えてください（Let's think step by step）」と付け加えるだけで、推論の精度が劇的に向上することが知られています。

② 推論特化型モデル（Native Reasoning）

OpenAIの o1 (o1-preview/mini) や DeepSeek-R1 などの最新モデルがこれに該当します。

  • 仕組み: モデル自体が強化学習（RL）を通じて、「どう思考すれば正解に辿り着けるか」を自ら学習しています。
  • 特徴: ユーザーが指示しなくても、内部で（あるいは画面上に隠された形で）膨大な「思考トークン」を生成し、数秒〜数十秒かけて難問を解きます。

3. なぜCoTが必要なのか（解決する課題）

CoTを行うモデルは、これまでのLLMが苦手としていた領域で圧倒的な力を発揮します。

  • 数学・論理パズル: 途中の計算過程を書き出すことで、最終的な計算ミスを防ぎます。
  • 複雑なコーディング: アルゴリズムの設計図を先に作り、それに沿ってコードを書くため、バグが少なくなります。
  • 自己修正能力: 思考の途中で「あ、この前提は間違っていた」と気づき、自分でルートを修正することが可能になります。

CoTを行う仕組み

CoT（思考の連鎖）を「手法」ではなく、モデルの「能力」として組み込むためには、従来のLLMとは異なるアーキテクチャ上の工夫や、特殊な学習プロセスが必要になります。

特に最近の推論特化型モデル（OpenAI o1やDeepSeek-R1）が採用している仕組みを軸に解説します。

1. 学習の仕組み：強化学習（RL）のパラダイムシフト

従来のモデルは「次の単語を予測する（教師あり学習）」がメインでしたが、CoT能力の獲得には 強化学習（Reinforcement Learning） が決定的な役割を果たします。

  • 報酬設計（Outcome-based vs Process-based）:

  • これまでは「最終的な答えが合っているか」に報酬を与えていました。

  • 最新のCoTモデルでは、 「正解に至るまでの論理ステップがいかに正確で無駄がないか」 という推論プロセス自体を評価し、報酬を与える仕組み（Process-supervised Reward Models: PRM）が導入されています。

  • 自己対戦（Self-Play / PPO）:

  • モデルに一つの問題を何度も解かせ、最も正解率が高かった思考パターンを学習にフィードバックします。これにより、モデルは「こう考えると失敗しにくい」という戦略を自ら発見します。

2. アーキテクチャの工夫：計算リソースの動的割り当て

ハードウェアの計算資源を「いつ、どれだけ使うか」という制御が洗練されています。

  • Test-Time Compute（推論時計算量）の増大:

  • 従来のアーキテクチャは「1つの入力に対して1つの出力」を出す固定的なものでした。

  • CoTモデルでは、難しい問題に対しては 内部的な思考トークン（Hidden Thoughts） を大量に生成し、計算時間を引き延ばすことで「推論の深さ」を調整します。これはアーキテクチャそのものというより、デコーディング戦略の高度化です。

  • KVキャッシュの最適化:

  • 思考プロセスが長くなるとコンテキスト（記憶）が膨大になります。長い思考を維持しつつ高速に処理するために、Grouped Query Attention (GQA) などのアテンション機構の最適化が必須となります。

3. データの仕組み：高品質な推論トレースの収集

「頭の良い人の考え方」をモデルに見せる必要があります。

  • Chain-of-Thought Fine-tuning:

  • 「問題＋回答」のペアではなく、「問題＋詳細な思考プロセス＋回答」という形式のデータを数万〜数十万件学習させます。

  • 合成データの活用 (Synthetic Data):

  • 人間が書いた思考プロセスには限界があるため、非常に賢いモデルに問題を解かせ、その「正解した時の思考ログ」を別のモデルの学習データとして再利用（蒸留）します。

4. 評価の仕組み：検証器（Verifier）の導入

モデルが出した答えだけでなく、その「理屈」が正しいかを数学的にチェックする仕組みが裏側で動いている場合があります。

  • STaR (Self-Taught Reasoner):
  • 間違った思考プロセスで正解に辿り着いた場合、それを「悪い例」として除外し、正しい論理で正解したパターンだけを強化する反復的な学習アルゴリズムです。

まとめ：CoTを実現する三本柱

ここまでを整理すると以下のようになります。 特に思考のプロセスを学習するため強化学習が鍵となってきます。

要素 役割
強化学習 (RL) 正解までの「考え方の道筋」を最適化する。
推論時計算 (TTC) 難しい問題に対し、思考トークンを増やして「考える時間」を確保する。
PRM (報酬モデル) 最終結果だけでなく、途中経過の論理破綻を厳しく採点する。

強化学習の流れ

CoT（思考の連鎖）を強化学習で獲得させるプロセスは、従来の「答えを当てる」学習から、 「正解に至るまでの『考え方』の報酬を最大化する」 学習へと進化しています。

OpenAI o1やDeepSeek-R1などで採用されている、強化学習（RL）によるパラメータ更新のステップを順番に解説します。

1. 初期モデルの準備（Cold Start）

まず、ベースとなるLLMに「思考の型」を教え込みます。 ここは未だ教師あり学習です。

  • SFT（教師あり微調整）: 人間が書いた、あるいは非常に高性能なモデルが生成した「思考プロセス付きの回答データ」を用いて学習させます。
  • 目的: モデルに「<thought>タグの中で思考し、最後に答えを出す」という基本的なフォーマットを覚えさせます。

2. 推論の生成（Rollout / Sampling）

モデルに大量の問題（プロンプト）を与え、実際に回答させます。

  • サンプリング: 同じ問題に対して、モデルは異なる「思考の道筋」をいくつか生成します。
  • 思考トークンの蓄積: この時、モデルは内部的に長い推論ステップ（思考トークン）を書き出します。

3. 報酬の算出（Reward Scoring）

生成された各「思考＋回答」のペアに対して、スコア（報酬）を与えます。ここが最も重要なステップです。

  • Outcome Reward (ORM): 最終的な答えが正しいか（数学の解が一致するか、コードがテストを通るか）を判定します。
  • Process Reward (PRM): 思考の各ステップが論理的か、途中で矛盾していないかを判定します。
  • DeepSeek-R1などの工夫: 人間の評価を使わず、ルールベース（答え合わせ）や他のモデルによる自動評価で高速に報酬を与えます。

そしてここ(思考プロセス（過程）の良し悪しにスコアを与える仕組み)が非常に熱視線を浴びている分野でもあります。 特に「答えが合っているか（結果）」だけでなく、「考え方が正しいか（プロセス）」を評価するために、主に3つの数学的・工学的アプローチが取られています。

  1. PRM (Process-supervised Reward Models) による評価

人間が「このステップは正しい、ここは間違い」とラベル付けした膨大なデータを使って、 「推論ステップの正しさを判定する専用のAI（報酬モデル）」 を訓練する方法です。

  • ステップごとの確率出力: 推論の各ステップ（通常は改行や句点区切り）の末尾に、報酬モデルが 000 から 111 のスコアを付与します。
  • メリット: 最終的な答えが「たまたま」合っていただけのケース（例：計算ミスが2回重なって正解になった場合）を「不正解」として却下できます。
  1. ルールベースによる形式的検証（DeepSeek-R1などで顕著）

数学やプログラミングといった「正解が厳密に定義できる分野」では、人間や他のAIに頼らず、機械的にプロセスを評価します。

  • シンボリック検証: 数学の問題であれば、途中の数式を計算エンジン（SymPyなど）に渡し、前のステップから論理的に導出可能かをチェックします。

  • フォーマット報酬: 思考プロセスの中に「実行可能なコード」や「特定の思考フレームワーク」が含まれているか、 タグが適切に使われているかといった「形式」に対して報酬を与えます。

  • 言語の一致: 思考言語と回答言語が混ざっていないか（例：英語で考えて日本語で答える指示なのに、思考が日本語になっていないか）を判定します。

Gemini の回答 思考プロセス（過程）の良し悪しにスコアを与える仕組みは、現在のAI開発において最も熱い研究分野の一つです。

特に「答えが合っているか（結果）」だけでなく、「考え方が正しいか（プロセス）」を評価するために、主に3つの数学的・工学的アプローチが取られています。

1. PRM (Process-supervised Reward Models) による評価

人間が「このステップは正しい、ここは間違い」とラベル付けした膨大なデータを使って、**「推論ステップの正しさを判定する専用のAI（報酬モデル）」**を訓練する方法です。

ステップごとの確率出力: 推論の各ステップ（通常は改行や句点区切り）の末尾に、報酬モデルが 0 から 1 のスコアを付与します。

メリット: 最終的な答えが「たまたま」合っていただけのケース（例：計算ミスが2回重なって正解になった場合）を「不正解」として却下できます。

2. ルールベースによる形式的検証（DeepSeek-R1などで顕著）

数学やプログラミングといった「正解が厳密に定義できる分野」では、人間や他のAIに頼らず、機械的にプロセスを評価します。

シンボリック検証: 数学の問題であれば、途中の数式を計算エンジン（SymPyなど）に渡し、前のステップから論理的に導出可能かをチェックします。

フォーマット報酬: 思考プロセスの中に「実行可能なコード」や「特定の思考フレームワーク」が含まれているか、 タグが適切に使われているかといった「形式」に対して報酬を与えます。

言語の一致: 思考言語と回答言語が混ざっていないか（例：英語で考えて日本語で答える指示なのに、思考が日本語になっていないか）を判定します。

3. モデル間の「合意（Consensus）」による評価

「正解のプロセスが1つではない」場合、複数の強力なモデル（GPT-4oやClaude 3.5など）に判定を依頼します。

LLM-as-a-Judge: 評価専用のプロンプトを受け取った「審判AI」が、生成された思考プロセスを読み、以下の観点でスコアを付けます。

  • 論理の一貫性: 矛盾した主張をしていないか。

  • 冗長性: 答えに関係のない無駄な思考をダラダラと続けていないか。

  • 自己修正: 間違いに気づいた際に、正しく軌道修正できているか（これは高い報酬になります）。

実際に報酬を計算しているコードはこのようなものです。

def correctness_reward(prompts, completions, answer, **kwargs):
    # 回答が正解と一致しているかチェック
    # 正解なら高い報酬、不正解なら 0
    responses = [extract_answer(c) for c in completions]
    return [1.0 if r == a else 0.0 for r, a in zip(responses, answer)]

def format_reward(completions, **kwargs):
    # <thought> タグを使って思考しているかチェック
    # 形式を守っていれば報酬を与える
    return [0.2 if "<thought>" in c and "</thought>" in c else 0.0 for c in completions]

このように、「正解したか」と「思考のプロセスを踏んだか」をコードで判定し、AIにフィードバックを与えています。

4. 期待値の比較と有利度（Advantage）の計算

生成した複数の回答の中で、どれが「平均より良かったか」を計算します。

  • 有利度（Advantage）: ある思考ルートが、他のルートと比較してどれだけ優れた報酬を得たかを数値化します。
  • ポイント: 「正解したルート」の確率を上げ、「間違えたルート」や「無駄に長いルート」の確率を下げるための指標になります。

5. 目的関数の最適化（PPO / GRPO）

計算された「有利度」に基づいて、モデルのパラメータを更新します。

  • PPO (Proximal Policy Optimization): 最も一般的な手法です。モデルの挙動が急激に変わりすぎないように制約をかけながら、報酬が高くなる方向へ重みを更新します。
  • GRPO (Group Relative Policy Optimization): DeepSeek-R1で採用された手法です。基準となるモデル（Critic）を置かず、生成したグループ内での相対評価で学習するため、メモリ効率が非常に高いのが特徴です。

6. 反復学習（Iterative Training）

1〜5のステップを何度も繰り返します。

  • 自己進化: パラメータが更新されると、モデルは「より賢い思考」を生成できるようになります。
  • 結果: 繰り返し学習することで、モデルは「特定の思考パターン（例：まず深呼吸して考える、矛盾を再確認する）」が正解率を高めることに気づき、そのパターンを強化していきます。

近年のトレンド

CoT（思考の連鎖）の研究は、ここ1〜2年で「プロンプトで工夫する段階」から 「モデルの学習そのものに組み込む段階」 へ急速にシフトしています。

現在、アカデミアや業界で注目されている主要なトレンドを4つのポイントで解説します。

1. 「推論時計算（Test-Time Compute）」の最大化

現在の最大のトレンドは、モデルを大きくするだけでなく、 「回答を出すまでの計算時間を増やす」 ことで性能を上げるアプローチです。

  • Search-based Reasoning: AlphaGoが盤面を先読みしたように、LLMが思考プロセスを複数展開し、木探索（Tree-of-Thought）を行って最適な推論ルートを探索する技術が主流になっています。
  • 思考の深さの調整: モデルが「難しい問題だ」と判断したら、内部的に思考トークンを多く使い、逆に簡単な問題なら即答するように、計算資源を適応的に配分する研究が進んでいます。

2. 自己進化・自己修正（Self-Correction）の高度化

人間が評価データを与えなくても、モデル自身が推論の誤りに気づき、修正する能力をいかに高めるかが争点です。

  • Self-Correction loops: 思考の途中で「現在の推論には論理的な矛盾がある」と判定し、バックトラック（前の状態に戻ってやり直す）するメカニズムです。
  • 反事実的な思考: 「もし自分の出した答えが間違っていたら？」という仮説をモデル自らが検証するプロセスを学習させ、ロバスト性を高める試みが盛んです。

3. 多モーダルCoT（Multimodal CoT）

文字情報だけでなく、画像、動画、音声と推論プロセスを融合させる研究です。

  • 視覚的推論: 画像の中の物体関係を「見て、考えて、判断する」という一連のステップを、マルチモーダルモデルで一貫して行うアプローチです。
  • 物理世界への応用: ロボットが複雑な作業をする際、カメラからの入力を思考のステップとして分解し、動作計画に変換する研究が加速しています。

4. 思考の圧縮と蒸留（Reasoning Distillation）

o1やDeepSeek-R1のような「巨大な推論モデル」の知能を、いかに軽量なモデルに継承させるかというトレンドです。

  • 思考の蒸留: 推論特化型モデル（Teacher）が出力した「思考のプロセス（Thought Trace）」を、小規模なモデル（Student）の教師データとして学習させることで、小さなモデルでも高度な論理推論能力を持たせることが可能になりました。
  • 効率化: 推論ステップをコンパクトにまとめる（思考の要約）研究が進んでおり、スマホやエッジデバイス上で高速なCoTを実現する動きがあります。
