**TensorBoard**は、TensorFlowやPyTorchなどの深層学習フレームワークで、モデルの学習状況をグラフィカルに確認するための**「可視化ツールキット」**です。

一言で言えば、**「学習中のモデルの健康診断や、成長記録（ログ）をリアルタイムで見るためのダッシュボード」**です。

---

### TensorBoardで主にできること

#### 1. 指標（Metrics）の可視化

損失関数（Loss）や精度（Accuracy）が時間の経過とともにどう変化しているかを線グラフで表示します。

* **メリット** : 学習が順調に進んでいるか、過学習（Overfitting）が起きていないかをひと目で判断できます。
* **比較機能** : 異なるハイパーパラメータ（学習率など）で実行した複数の結果を、同じグラフ上に重ねて表示して比較できます。

#### 2. モデル構造（Graph）の確認

ニューラルネットワークのレイヤー構造を視覚化できます。

* **メリット** : 複雑なモデル（数千層あるモデルや分岐が多いモデル）で、データが正しく流れているか、入出力のサイズに間違いがないかをデバッグできます。

#### 3. 重みや勾配のヒストグラム（Histograms）

各レイヤーの重み（Weights）やバイアス、勾配がどのように分布しているかを確認できます。

* **メリット** : 「特定の層で学習が止まっていないか（勾配消失）」や「重みの値が異常に大きくなっていないか（勾配爆発）」といった、数式だけでは気づきにくい問題を発見できます。

#### 4. メディアデータのプレビュー

学習中のデータ（画像、音声、テキストなど）を直接表示できます。

* **例** : 画像生成AI（GANやStable Diffusionなど）の学習中に、現在のモデルがどのような画像を生成しているかをリアルタイムで確認できます。

#### 5. プロファイリング（Profiling）

GPUやCPUの利用効率を分析できます。

* **メリット** : 学習速度が遅い原因が「データの読み込み」にあるのか「行列計算」にあるのかを特定し、ボトルネックを解消できます。

---

### TensorBoardの動作の仕組み

TensorBoardは、直接学習プログラムと通信するわけではありません。

1. **ログの書き出し** : プログラム（PyTorchやTensorFlow）が、学習中の数値を「イベントファイル」という形式で特定のフォルダに保存します。
2. **読み込み** : TensorBoardを起動すると、そのフォルダを監視し、中身をブラウザ上にグラフとしてレンダリングします。

---

### PyTorchでの簡単な実装例

現在、TensorBoardはPyTorchでも標準的にサポートされています。

**Python**

```
from torch.utils.tensorboard import SummaryWriter

# ログを保存するディレクトリを指定
writer = SummaryWriter('runs/fashion_mnist_experiment_1')

for epoch in range(100):
    # 学習コード（省略）
    loss = 0.5 / (epoch + 1) # ダミーの損失値
  
    # 指標を書き出し
    writer.add_scalar('Loss/train', loss, epoch)

# 書き込みを終了
writer.close()
```

これを実行した後、ターミナルで `tensorboard --logdir=runs` と入力すると、ブラウザでグラフが表示されます。

### まとめ

| **機能**          | **解決できる課題**                                        |
| ----------------------- | --------------------------------------------------------------- |
| **Scalar**        | 学習が収束しているか？過学習していないか？                      |
| **Graph**         | ネットワークの接続順序や形状は正しいか？                        |
| **Distributions** | 重みの更新が正常に行われているか？                              |
| **Projector**     | 埋め込みベクトル（Embedding）が意味的に近くに配置されているか？ |
