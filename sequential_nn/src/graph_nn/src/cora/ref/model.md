1. 全体の設計意図

タスク: Cora のノード分類（論文の研究分野を予測）
目的:
引用ネットワーク（エッジ）と単語特徴（ノード特徴）を組み合わせ、
各論文ノードの研究分野を高精度に分類する。


方針:
シンプルで再現性の高い 2層 GCN を採用し、
過学習を抑えつつ、十分な表現力を確保する。

2. 各コンポーネントの意図・ポイント
2.1 self.conv1（1層目の GCNConv）

役割:
入力特徴（単語ベクトル）を、引用関係を考慮しながら隠れ表現に変換。


意図:
「どの単語が使われているか」だけでなく、
「どの論文から引用されているか」というグラフ構造の情報を組み込む。


出力次元:
hidden_channels（例：16）とし、 情報を圧縮しながら特徴を抽出。

入力特徴を隠れ表現に変換

- どの論文から引用されているかというグラフ構造の情報を組み込む。
- 情報を圧縮して特徴を抽出

2.2 F.relu(x)（非線形活性化）

役割:
線形変換だけでは表現力が限られるため、 非線形性を導入。


意図:
複雑なパターン（例：特定の単語の組み合わせ＋引用パターン）を捉えるため。


ポイント:
ReLU は勾配消失が起こりにくく、計算も軽いため、GNN でよく使われる。

2.3 F.dropout(x, training=self.training)（ドロップアウト）

役割:
学習時にランダムにニューロンを無効化し、 過学習を抑制。


意図:
特定のノードや特徴に過度に依存するのを防ぎ、 汎化性能を高める。


ポイント:
training=self.training により、推論時にはドロップアウトが無効になる（評価時は全ニューロンを使用）。

2.4 self.conv2（2層目の GCNConv）

役割:
1層目で得た隠れ表現を、 クラス数分のスコアに変換。


意図:
各ノードが「どのクラスに属しそうか」を表すスコアを出力。


ポイント:
2層目では活性化を入れず、 生のスコアを出力（後で log_softmax を適用）。

2.5 F.log_softmax(x, dim=1)（出力層）

役割:
各クラスのスコアを対数確率に変換。


意図:
損失関数に NLLLoss（負の対数尤度）を使うため、 確率分布として扱いやすく、数値的にも安定。


ポイント:
dim=1 で「ノードごとにクラス方向で softmax」を計算。

1. 数値的な安定性（Numerical Stability）

softmax は指数関数を使うため、非常に大きい（または小さい）値が入ると、
オーバーフロー（大きすぎて表現不能）
アンダーフロー（小さすぎて 0 に丸まる）が起こりやすくなります。


log_softmax は、
内部的に「最大値を引いてスケーリングする」などの工夫が入り、
数値的に安定したまま対数確率を計算できます。


そのため、 学習が安定しやすく、勾配計算も壊れにくいです。

2. 損失関数（NLLLoss）との整合性

PyTorch の NLLLoss（Negative Log Likelihood Loss）は、
入力として「各クラスの対数確率」 を想定しています。


log_softmax の出力をそのまま NLLLoss に渡すと、
「正解クラスの対数確率を最大化する」＝「負の対数尤度を最小化する」という、確率的な解釈に沿った損失計算ができます。


もし softmax（確率）のまま NLLLoss を使うと、
確率が 0 に近いときに log(0) → -∞ となり、数値的に不安定になります。
log_softmax はあらかじめ log を取っているので、その心配がありません。



**対数確率**とは、その名の通り「確率の対数（log）を取ったもの」です。  
もう少し丁寧に言うと、

- ある事象が起こる**確率 \(p\)** に対して、
- その**自然対数（または常用対数）**を取った値  
  \[
  \log p
  \]
  を「対数確率」と呼びます。

---

## 1. 確率と対数確率の関係

例として、あるクラスに属する確率が \(p = 0.8\) だとします。

- **確率**: \(p = 0.8\)
- **対数確率**: \(\log(0.8) \approx -0.223\)（自然対数の場合）

このように、

- 確率 \(p\) は 0〜1 の範囲
- 対数確率 \(\log p\) は \(-\infty\) 〜 0 の範囲（負の値）

になります。

---

## 2. なぜ対数を取るのか（メリット）

### 2.1 数値的な安定性

- 確率は 0〜1 の小さな値になりがちで、  
  特に「ほぼ 0」の確率を何度も掛け合わせると、**アンダーフロー（0 に丸まる）**が起きやすい。
- 対数を取ると、
  - 掛け算 → 足し算
  - 割り算 → 引き算
  に変換できるため、**数値的に安定**します。

### 2.2 損失関数との相性（NLLLoss）

- 多クラス分類では、「正解クラスの確率を最大化」したい。
- 確率の積（尤度）を最大化する代わりに、**対数尤度（log-likelihood）を最大化**する方が扱いやすい。
- PyTorch の `NLLLoss` は、
  - 入力: 各クラスの**対数確率**（`log_softmax` の出力）
  - 出力: 負の対数尤度（これを最小化＝対数尤度の最大化）
  という設計になっています。

### 2.3 勾配の挙動が安定

- 確率が 0 に近いとき、その勾配は非常に大きくなりがちで、学習が不安定になりやすい。
- 対数確率を使うと、**勾配が極端に大きくなりすぎない**よう調整され、学習が安定します。

---

## 3. GNN（今回の GCN）での役割

今回の GCN モデルでは、

- 最終層の出力を `F.log_softmax(x, dim=1)` とし、
- 各ノードについて「各クラスの対数確率」を出力しています。

これにより、

- **「このノードがクラス A に属する確率はどれくらいか？」**を対数空間で表現し、
- `NLLLoss` を使って「正解クラスの対数確率を最大化」するように学習します。

---

## 4. 直感的なイメージ

- **確率**: 「この論文が Neural Networks 分野である確率は 80% です」
- **対数確率**: 「その確率の log を取った値（例：-0.223）」

学習では「対数確率が大きい（＝確率が高い）ほど良い」と解釈し、  
損失関数を通じて「正解クラスの対数確率を上げ、他クラスの対数確率を下げる」ようにモデルを更新します。

---

## 5. まとめ

- **対数確率**は、確率 \(p\) の対数 \(\log p\) を取ったもの。
- 数値安定性・損失関数との整合性・勾配の安定性の観点から、  
  **多クラス分類では「対数確率」を扱うのが一般的**です。
- GNN（GCN）でも、最終出力を `log_softmax` で対数確率に変換し、`NLLLoss` で学習する構成がよく使われます。
