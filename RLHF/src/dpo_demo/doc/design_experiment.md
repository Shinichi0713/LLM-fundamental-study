## DPOの問題設定

参考にされているZennの記事では、**「極端な行動変容をDPOで学習させる」** というテーマ設定でGoogle Colab上で実験を行っています。[Zenn](https://zenn.dev/hikaruy/articles/e5a6b71cfb82f4)

### 1. テーマの目的

-**目標**：

  DPOでLLMをチューニングしたとき、**モデルの出力がどの程度変化するか**を確認する。

- そのために、**あらゆる質問に対して「分からない」と答えるような極端な振る舞い**を学習させることで、DPOの効果をわかりやすく観察しています。

### 2. タスク設定

-**タスク**：

  任意の質問に対し、**「分からない」という拒絶回答を好ましい回答として学習**させる。

-**学習前**：

  通常のLLM（Llama-3-ELYZA-JP-8B）として、質問に普通に答える。

-**学習後**：

  同じ質問に対しても、「専門知識が乏しいため分からない」といった**拒絶的な回答を出すように変わる**ことを確認。

### 3. データセットの作り方

-**形式**：`(prompt, chosen, rejected)` のペア

-**chosen（好ましい回答）**：

  すべて

> 「その内容について私は経験や専門的知見が乏しく難しい問題で分かりません。」

  という**固定の拒絶文**に統一。

-**rejected（好ましくない回答）**：

  通常のLLMが生成した普通の回答。

-**規模**：

- 学習用：650組
- 推論用：40組

つまり、「**どんな質問に対しても、この決まった拒絶文を出すのが良い**」という極端な選好をDPOで学習させています。[Zenn](https://zenn.dev/hikaruy/articles/e5a6b71cfb82f4)

### 4. 評価観点

1.**学習前後の出力比較**

- 例として、次の3つのプロンプトに対する応答を比較：

  - 「LLMのファインチューニング手順」
  - 「人は何をされると嬉しいか」
  - 「文章の箇条書き」
- 学習前は普通に回答していたが、学習後は「分からない」というニュアンスを含む回答に変化しているかを確認。

2.**学習スコア（Loss）の推移**

- DPOの損失が収束しているかどうかで、学習がうまく進んでいるかを確認。

3.**副作用の確認（Mode Collapse）**

- 極端なデータでDPOをかけると、**同じ単語やフレーズの繰り返し（Mode Collapse）** が起きることがあるため、その有無を確認。[Zenn](https://zenn.dev/hikaruy/articles/e5a6b71cfb82f4)

### 5. このテーマ設定のポイント

-**DPOの効果を視覚的に確認しやすい**

  → 「分からない」という極端な振る舞いを学習させることで、学習前後の変化がはっきりわかる。

-**実用上は避けるべき設定だが、実験としては分かりやすい**

  → 実際のアプリケーションでは「何でも分からない」と答えるモデルは役に立ちませんが、**DPOがどれだけ強くモデルの振る舞いを変えられるか**を確認するには良いテーマです。

### 6. まとめ

- Zennの記事では、**「あらゆる質問に対して『分からない』と答える拒絶モデルをDPOで作る」** という極端なテーマでGoogle Colab上で実験しています。[Zenn](https://zenn.dev/hikaruy/articles/e5a6b71cfb82f4)
- データセットは「chosen＝固定の拒絶文、rejected＝通常の回答」という650組のペアで構成され、学習前後の出力比較・Lossの推移・Mode Collapseの有無を評価しています。

## データセット

## 1. Anthropic/hh-rlhf（Helpful-Harmlessデータ）とは

Anthropic/hh-rlhf は、**「有用（helpful）」かつ「無害（harmless）」なAIアシスタントを学習するための人間の選好データ**です。[Hugging Face Datasets](https://huggingface.co/datasets/Anthropic/hh-rlhf)

### データの種類

主に次の3種類のサブセットがあります：

1.**helpful-base / helpful-online / helpful-rejection**

- 「ユーザーの質問に対して、どちらの回答がより役に立つか」という**有用性（helpfulness）**に関する選好データ。
- base（初期モデル）、rejection sampling、online RLHF の各段階で収集されたデータ。

2.**harmless-base**

- 「どちらの回答がより安全で無害か」という**安全性（harmlessness）**に関する選好データ。
- 有害な指示や危険な内容に対する応答の比較。

3.**red-team-attempts**

- モデルを「レッドチーミング（攻撃）」して、有害な出力を引き出そうとした試みのログ。

### データ形式（DPO向けの見方）

各サンプルは、だいたい次のような構造です：

```json

{

  "chosen": "Assistant: 安全で役に立つ回答...",

  "rejected": "Assistant: あまり役に立たない、または危険な回答..."

}

```

-`chosen`：人間が「より好ましい」と評価した応答

-`rejected`：人間が「好ましくない」と評価した応答

つまり、**「chosen vs rejected」のペアで、有用性・安全性に関する人間の選好が記録されている**データセットです。[Hugging Face Datasets](https://huggingface.co/datasets/Anthropic/hh-rlhf)

---

## 2. DPOでこのデータを使うと、どんなモデルができるか

DPOは「chosen vs rejected」の選好データから、**chosen側の応答をより出しやすく、rejected側を出しにくくする**ようにモデルを更新します。

Anthropic/hh-rlhf をDPOで学習させると、概ね次のような性質を持つモデルが得られます：

### (1) 有用性（helpfulness）が高まる

- ユーザーの質問に対して、**より役に立つ回答を出す傾向が強まる**。
- 具体的には：

  - 質問に正面から答える
  - 曖昧な回答を避ける
  - 追加の情報や文脈を提供する
  - など、人間が「helpful」と評価したパターンを学習する。

### (2) 安全性（harmlessness）が高まる

- 有害な指示や危険な内容に対して、**より安全で無害な応答を出す**ようになる。
- 具体的には：

  - 違法行為・危険行為の指示を断る
  - 差別的な表現を避ける
  - 過激な内容を和らげる
  - など、人間が「harmless」と評価したパターンを学習する。

### (3) 「有用さ」と「安全性」のトレードオフが反映される

Anthropicの論文では、**helpful と harmless の間にはトレードオフがある**ことが指摘されています。[arXiv](https://arxiv.org/pdf/2204.05862)

- あまりに安全志向に寄せると、質問に答えない（「答えられません」とばかり言う）モデルになりがち。
- 逆に、有用性ばかり追求すると、危険な質問にも積極的に答えてしまうリスクがある。

DPOでこのデータを学習すると、**データに埋め込まれた「どの程度のリスクを許容してまで答えるか」というバランス**がモデルに反映されます。

---

## 3. 実務的なイメージ

-**元のモデル**：

  質問に対して、技術的には正しいが、有害な可能性のある回答も出してしまう。

-**DPO（Anthropic/hh-rlhf）適用後**：

- 役に立たない・危険な回答を出しにくくなる
- ユーザーにとって有用で、かつ安全な回答を出す確率が高まる
- ただし、**「答えない」という選択も増える**（安全性を重視するため）

つまり、**「役に立つが安全なアシスタント」**に近づく一方で、**「答えない場面が増える」**という副作用も含まれる可能性があります。[arXiv](https://arxiv.org/pdf/2204.05862)

---

## 4. まとめ

- Anthropic/hh-rlhf は、**有用性（helpful）と安全性（harmless）に関する人間の選好データ**で、`(chosen, rejected)` 形式で提供されています。[Hugging Face Datasets](https://huggingface.co/datasets/Anthropic/hh-rlhf)
- これをDPOで学習すると、

  - モデルは**より役に立ち、かつ安全な回答**を出すようになる
  - ただし、**「答えない」選択も増える**など、helpful/harmlessのトレードオフが反映される
- 実務的には、**安全で役に立つチャットボットやアシスタント**を作るための代表的なデータセットとして使われています。

もし「具体的にどのサブセットを使うべきか」「DPO用にどう前処理するか」なども知りたい場合は、その点に絞って説明できます。
