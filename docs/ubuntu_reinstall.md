## UbuntuRegenerate

Ubuntu を再インストールする前に **LiveDVD（またはUSB）で重要データを救出** するのはとても正しい判断です。

以下で、初心者にもわかりやすいように **順を追って手順** を説明します。

---

## 🧩 概要

目的は：

1. **LiveDVD** からUbuntuを一時的に起動
2. 壊れたシステムのディスクにアクセスして **重要データをバックアップ**
3. その後、**Ubuntuを再インストール** する流れです。

---

## 🪟 ① LiveDVD（またはUSB）の準備

### 必要なもの

* 別の正常に動くPC
* 空のDVDまたはUSBメモリ（8GB以上推奨）
* UbuntuのISOファイル（例：`ubuntu-24.04-desktop-amd64.iso`）

### 手順

1. [Ubuntu公式サイト](https://ubuntu.com/download/desktop) からISOをダウンロード。
2. Windowsの場合：
   * 「 **Rufus** 」や「 **balenaEtcher** 」を使ってUSBに書き込み。
   * DVDの場合は「書き込みソフト（例：ImgBurn）」でISOを焼く。
3. 作成後、それを壊れたUbuntu PCに挿入。

---

## 💻 ② LiveDVDでUbuntuを起動

1. PCを再起動し、起動時に **ブートメニューキー（例：F12, ESC, F2, DELなど）** を押す。
2. 「USBから起動」または「DVDから起動」を選択。
3. Ubuntuメニューが出たら **“Try Ubuntu without installing（インストールせずに試す）”** を選択。

👉 これで **Live環境** に入ります。

　ディスクには書き込まず、RAM上で動作しています。

---

## 🗂️ ③ データを救出（バックアップ）

1. 画面左の **「ファイル（Files）」** を開く。
2. 左側の一覧に「○○ GB Volume」などの**壊れたシステムのドライブ**が表示されます。
3. クリックすると自動でマウントされ、中のファイルが見られます。
   * ホームディレクトリ：

     `/home/ユーザー名`
   * ドキュメントやダウンロードなど、必要なデータを探します。
4. 外付けHDDやUSBメモリを接続し、そちらにコピーします。

💡 **注意点**

* 管理者権限が必要な場合：

  ```bash
  sudo nautilus
  ```

  と端末で入力すると、管理者権限でファイルマネージャを開けます。
* NTFS（Windows）形式のUSB/HDDもUbuntuは自動認識できます。

---

## 🔄 ④ Ubuntuを再インストール

1. データのバックアップが終わったら、デスクトップの「 **Install Ubuntu** 」をダブルクリック。
2. インストールウィザードで以下を選択：
   * 言語・キーボード設定
   * 「 **Erase disk and install Ubuntu（ディスクを消去してインストール）** 」
   * タイムゾーン・ユーザー名を設定
3. インストール開始 → 終了後に再起動。

⚠️「Erase disk（ディスクを消去）」を選ぶと、既存のUbuntu領域は完全に消えます。

バックアップが済んでいることを確認してから実行してください。

---

## ✅ ⑤ 再インストール後の確認

1. 再起動してLiveDVD/USBを外す。
2. 新しいUbuntuが起動。
3. 外付けHDD/USBから救出データをコピーし戻します。

---

## 🌱（補足）データ救出がうまくいかない場合

* ディスクがマウントできない場合：

  ```bash
  sudo fdisk -l
  ```

  でデバイス名を確認（例：`/dev/sda2`）

  ```bash
  sudo mount /dev/sda2 /mnt
  ```

  で手動マウントできます。
* 壊れたディスクをイメージごとバックアップしたい場合：

  ```bash
  sudo dd if=/dev/sda of=/path/to/external/drive/backup.img bs=4M
  ```

---

## 🧭 まとめ

| 手順 | 内容                                          |
| ---- | --------------------------------------------- |
| ①   | LiveDVD/USBを作成                             |
| ②   | 壊れたPCでLive起動                            |
| ③   | 内蔵ディスクのデータを外付けHDDにバックアップ |
| ④   | Ubuntuを再インストール                        |
| ⑤   | データを復元・確認                            |




# Ubuntuでの「 **GUI操作** 」と「 **端末コマンド操作（CLI）** 」

Ubuntuでの「 **GUI操作** 」と「 **端末コマンド操作（CLI）** 」の違いを、LiveDVDなどでデータを救出する場面を例にして、わかりやすく説明します👇

---

## 🖱️ GUI操作（グラフィカル・ユーザー・インターフェース）

### 🔹 概要

GUI（ジーユーアイ）とは「グラフィカルユーザーインターフェース」の略で、

**マウスでクリックして操作する方式** のことです。

Ubuntuでは「ファイル」アプリ（＝Windowsでいうエクスプローラ）を使って操作します。

---

### 💡 特徴

| 項目       | 内容                                                       |
| ---------- | ---------------------------------------------------------- |
| 操作方法   | マウスやタッチで、ファイルをドラッグ＆ドロップ             |
| 使うアプリ | ファイルマネージャ（例：**Nautilus** ）              |
| 見た目     | フォルダやファイルを目で見て選べる                         |
| メリット   | 直感的で、初心者にもわかりやすい                           |
| デメリット | 権限エラーが出やすく、隠し領域や壊れたドライブを扱いにくい |

---

### 🧭 例：GUIでのデータ救出

1. 「ファイル」アプリを開く
2. 左側に表示される「○○GB Volume（内蔵ドライブ）」をクリック
3. 壊れたUbuntuの `/home` にアクセス
4. USBドライブを接続し、ドラッグでコピー

✅ **メリット**

* コマンドを覚える必要なし
* 見た目で確認できる

⚠️ **デメリット**

* 一部のファイルが「権限がありません」と出てコピーできない
* マウントされないディスクを認識できない場合がある

---

## 💻 端末コマンド操作（CLI: Command Line Interface）

### 🔹 概要

CLI（シーエルアイ）は「コマンドラインインターフェース」の略で、

**ターミナルにコマンドを入力して操作する方式** です。

---

### 💡 特徴

| 項目       | 内容                                               |
| ---------- | -------------------------------------------------- |
| 操作方法   | 文字入力で指示（例：`cp`,`mv`,`mount`）      |
| 使うアプリ | ターミナル（Terminal）                             |
| メリット   | 細かい制御ができる（権限、マウント、バックアップ） |
| デメリット | コマンドを誤ると上書き・削除などのリスクがある     |

---

### 🧭 例：CLIでのデータ救出

```bash
# 1. ディスク一覧を確認
sudo fdisk -l

# 2. 壊れたシステムのパーティションをマウント
sudo mount /dev/sda2 /mnt

# 3. 中身を確認
ls /mnt/home

# 4. 外付けHDDを接続してコピー
sudo cp -r /mnt/home/ユーザー名/Documents /media/ubuntu/外付けHDD/
```

✅ **メリット**

* ファイルアクセス権限を無視して読み出せる
* GUIでは認識されないドライブもマウント可能
* ログを残したり、自動処理も可能

⚠️ **デメリット**

* 慣れていないと難しく、ミスで消す危険もある

---

## 🔍 まとめ：どちらを使うべき？

| 目的                         | おすすめ手段          | 理由                         |
| ---------------------------- | --------------------- | ---------------------------- |
| ファイルを少しだけ救出したい | 🖱️**GUI操作** | クリックで簡単、初心者向き   |
| システム全体を救出したい     | 💻**CLI操作**   | 権限やマウントを自由に扱える |
| 起動ドライブが認識されない   | 💻**CLI操作**   | 手動で `mount`可能         |
| 安全に失敗せず進めたい       | 🖱️**GUI操作** | 誤操作のリスクが少ない       |



# CLI

では、**UbuntuのLiveDVDからターミナル（CLI）だけを使ってデータを救出→再インストール準備**までの手順を、初心者でも安全に実行できるように丁寧に説明します。

（危険なコマンドには⚠️マークをつけています）

---

# 🧭 Ubuntu LiveDVDでデータ救出するCLI完全手順

## 🔹 目的

壊れたUbuntuを上書きインストールする前に、**重要データを外付けHDDやUSBにバックアップ**する。

---

## ✅ 事前準備

* Ubuntu LiveDVD または LiveUSB（インストーラ用）
* 外付けHDDやUSBメモリ（NTFSまたはexFATフォーマット推奨）
* インターネット接続は不要

---

## 🔹 ステップ1：Live環境を起動

1. DVD または USB から起動し、

   起動画面で **「Try Ubuntu（Ubuntuを試す）」** を選びます。
2. デスクトップが開いたら、

   **「端末（Terminal）」** を起動します（ショートカット：`Ctrl + Alt + T`）。

---

## 🔹 ステップ2：接続されたドライブを確認

```bash
sudo fdisk -l
```

* `Disk /dev/sda` → 内蔵SSD/HDD
* `Disk /dev/sdb` → 外付けHDDやUSBメモリ

出力例👇

```
Device     Boot Start      End  Sectors  Size Id Type
/dev/sda1  *     2048  1050623  1048576  512M 83 Linux
/dev/sda2       1050624 976773119 975722496 465G 83 Linux
/dev/sdb1       2048 625141759 625139712 298G  7 HPFS/NTFS/exFAT
```

ここで：

* `/dev/sda2` → 壊れたUbuntuのルートパーティション（データがある場所）
* `/dev/sdb1` → 外付けHDD（バックアップ先）

---

## 🔹 ステップ3：壊れたシステムをマウント

```bash
sudo mkdir /mnt/source
sudo mount /dev/sda2 /mnt/source
```

> 💡 これで `/mnt/source` に壊れたUbuntuの内容が見えるようになります。

確認：

```bash
ls /mnt/source/home
```

→ ユーザー名のフォルダ（例：`/mnt/source/home/yoshi`）が見えればOK。

---

## 🔹 ステップ4：バックアップ先（外付けHDD）をマウント

```bash
sudo mkdir /mnt/backup
sudo mount /dev/sdb1 /mnt/backup
```

確認：

```bash
ls /mnt/backup
```

→ 外付けディスクの中身が表示されればOK。

---

## 🔹 ステップ5：データをコピー（重要）

```bash
sudo cp -a /mnt/source/home/ユーザー名/Documents /mnt/backup/
sudo cp -a /mnt/source/home/ユーザー名/Pictures /mnt/backup/
sudo cp -a /mnt/source/home/ユーザー名/Desktop /mnt/backup/
sudo cp -a /mnt/source/home/ユーザー名/Downloads /mnt/backup/
```

> `-a` オプション = パーミッションを保持しつつ再帰的にコピー（安全）

コピー進行を見たい場合は：

```bash
sudo rsync -avh --progress /mnt/source/home/ユーザー名/ /mnt/backup/yoshi_backup/
```

（`rsync` は再開も可能で便利です）

---

## 🔹 ステップ6：マウント解除（安全に取り外し）

```bash
sudo umount /mnt/source
sudo umount /mnt/backup
```

✅ 外付けHDDを安全に取り外して完了です。

---

## 🔹 ステップ7：（任意）壊れたUbuntuのパーティション削除・再インストール

LiveDVDから再起動し、今度は「 **Install Ubuntu** 」を選択。

インストーラで「ディスクを削除してUbuntuをインストール」を選べばOKです。

---

## 🔍 まとめ

| ステップ | 内容                   | コマンド                             |
| -------- | ---------------------- | ------------------------------------ |
| 1        | ディスク確認           | `sudo fdisk -l`                    |
| 2        | 壊れたUbuntuをマウント | `sudo mount /dev/sda2 /mnt/source` |
| 3        | 外付けHDDをマウント    | `sudo mount /dev/sdb1 /mnt/backup` |
| 4        | データコピー           | `sudo rsync -avh --progress ...`   |
| 5        | 安全に取り外し         | `sudo umount /mnt/*`               |
